Classification map

You classify your content by adding relationships between content topics and subject topics.

Description

You can add classifications to your existing navigation map by nesting references to subject topics inside the existing references to content topics. You use the topicsubject element to refer to subject topics. You can also classify sets of topics using a topicSubjectTable element.

Example

The following example indicates that

  • The websecure.dita topic is about the Web Server and Security subjects.
  • The loginsetup.dita topic is about the Configuring, Web Server, and Security subjects. 
<topicref href="websecure.xml">
    <topicsubject>
        <subjectref href="webserver.xml"/>
        <subjectref href="security.xml"/>
    </topicsubject>
    <topicref href="https_protocol.xml"/>
    ... other subordinate content topics ...
</topicref>
<topicref href="loginsetup.xml" collection-type="sequence">
    <topicsubject>
        <subjectref href="configuring.xml"/>
        <subjectref href="webserver.xml"/>
        <subjectref href="security.xml"/>
    </topicsubject>
    <topicref href="editinguserdef.xml"/>
    ... other subordinate content topics ...
</topicref>
  • subjectCell
    Specifies subjects for a constrained classification of content topics.
  • subjectref
    Identifies a secondary subject covered by a content topic.
  • topicCell
    Contains one or more content topics that are classified by the subjects in the row.
  • topicsubject
    Identifies the subject matter of content topics.
  • topicSubjectHeader
    Specifies constraints on the subjects used in classifications.
  • topicSubjectRow
    Classifies one or more topics with a structured set of subjects.
  • topicSubjectTable
    Specifies a structured classification of content topics.
