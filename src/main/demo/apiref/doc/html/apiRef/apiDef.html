apiDef

The <apiDef> element provides the most general representation of the declared syntax.

Description

In the <apiDef> element, the <state>, <keyword>, and <ph> elements are provided to support this specialization.

Inheritance

topic/section reference/section

  • apiQualifier
    The <apiQualifier> element specifies a semantic flag that qualifies the interface.
  • apiRelation
    The <apiRelation> element specifies a related API item that is part of the declared interface to this API item.
  • apiType
    The <apiType> element specifies a type defined by the language itself that is part of the declared interface to this API item.
  • apiArray
    The <apiArray> element specifies any type parts, classifier or data within an interface where required.
  • apiData
    The <apiData> element specifies the actual value such as the initial value of variable, the default value of a parameter, or the value of a constant.
  • apiDefNote
    The <apiDefNote> element provides an optional note element to expand the information about a part of the declared interface for the API item within the <apiDef> element.
  • apiDefItem
    The <apiDefItem> element provides the most general definition of an API item when an API interface has subordinate structure such as parameters of a method or subfields of a record value.
  • apiItemName
    The <apiItemName> element provides the name for substructure of an interface such as parameter of an operation or subfield of a method.
